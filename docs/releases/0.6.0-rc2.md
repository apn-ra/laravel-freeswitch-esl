# 0.6.0 RC2 Release Notes

## Release posture

`apntalk/laravel-freeswitch-esl` `0.6.0-rc2` is the truthful next release
candidate for the `0.6.x` hardening line.

The repository has moved beyond the recorded `0.6.0-rc1` surface through
release-truth alignment, additional Laravel-boundary defensive proof, and
assignment-model persistence hardening. Because those changes were not part of
the originally recorded RC1 evidence set, the current repository state should
be treated as `0.6.0-rc2`, not promoted as unchanged RC1 and not claimed as
final `0.6.0` without fresh live validation for this exact target.

This release candidate now includes, in addition to the RC1 surface:

- corrected release-truth wording for the shipped `apntalk/esl-core` `^0.2.8`
  dependency floor in `CHANGELOG.md`
- tighter build-artifact hygiene so generated live-smoke review output remains
  outside tracked release noise
- explicit documentation that bounded health snapshot persistence stays rooted
  in `pbx_nodes` plus additive `settings_json` projection, without a dedicated
  health-history table in the current `0.6.x` scope
- explicit documentation that outbound server mode is deferred beyond the
  current `1.0.0` stabilization target and is not part of the current public
  API
- explicit documentation that the shipped Laravel event layer is the wrapper
  event surface, while higher-level Laravel-native normalized domain events
  remain deferred
- an integration proof that malformed decoded event payloads are ignored at the
  Laravel event-bridge boundary and truncated raw frames fail closed without
  dispatching Laravel events
- equivalent persistence-layer protection for
  `worker_assignments.assignment_mode`
- live-smoke workflow parity with the documented private validation wrapper and
  retained artifact layout

This release candidate still does not include:

- Laravel-owned reconnect/backoff mechanics
- Laravel-owned heartbeat/session lifecycle ownership
- Laravel-owned replay execution or session recovery
- protocol parsing or low-level transport ownership
- outbound server-mode support
- `1.0.0` stability claims
