# 0.6.0 Release Notes

`0.6.0` closes the bounded `0.6.x` hardening line for
`apntalk/laravel-freeswitch-esl` without widening package ownership.

The release keeps the current upstream runtime and replay boundaries intact
while tightening release truth, wiring confidence, and operator-facing
guidance.

## Highlights

- restored PHPStan coverage for the `FreeSwitchEslServiceProvider` wiring
  surface
- projected bounded `connect_timeout_seconds`,
  `stream_context_options.socket`, and `stream_context_options.ssl` settings
  onto the real `apntalk/esl-react` live connector path
- added explicit operator warnings when
  `freeswitch-esl.runtime.runner=non-live` is selected
- documented the Laravel bridge-wrapper `SCHEMA_VERSION` compatibility policy
- clarified that `build/live-smoke-review/` is ephemeral operator evidence, not
  durable repository truth

## Included in this release

- the documented `apntalk/esl-core` `^0.2.8` dependency floor reflected in
  release truth
- tighter build-artifact hygiene for retained live-validation output
- explicit documentation of the shipped health projection, outbound-support
  deferral, and normalized-event deferral posture
- Laravel-boundary defensive integration proof for malformed inbound handling
- persistence-layer protection for `worker_assignments.assignment_mode`
- live-validation workflow parity with the documented wrapper and retained
  artifact layout

## Deferred beyond this bounded release

- Laravel-owned reconnect/backoff lifecycle ownership
- Laravel-owned replay execution or runtime session recovery
- outbound server-mode support
- Laravel-native normalized domain events as a separate public layer
- replay backend expansion beyond the current bounded shipped set
- `1.0.0` stability guarantees
