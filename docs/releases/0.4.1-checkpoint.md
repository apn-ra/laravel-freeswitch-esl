# 0.4.1 Checkpoint Release Notes

## Release posture

`apntalk/laravel-freeswitch-esl` `0.4.1` is a truthful Laravel-side runtime observation checkpoint.
It extends the existing `0.4.0` lifecycle-observation milestone with richer prepared dial-target handoff into `apntalk/esl-react`.
It is not a Laravel-owned live runtime milestone.

This checkpoint now includes:
- Laravel/control-plane foundations
- partial `apntalk/esl-core` integration
- real `apntalk/esl-react` runner binding
- real upstream lifecycle snapshot observation through `RuntimeRunnerHandle::lifecycleSnapshot()`
- explicit prepared dial-target handoff through the Laravel-to-`apntalk/esl-react` adapter, including `tls://host:port` targets when the resolved `ConnectionContext` requires them

This checkpoint still does not include:
- direct `apntalk/esl-core` transport polling handoff
- Laravel-owned runtime supervision
- Laravel-owned reconnect/backoff ownership
- Laravel-owned heartbeat/session lifecycle ownership
- live TLS validation
- replay runtime orchestration

## Highlights

- Bumped the minimum supported `apntalk/esl-react` line to `^0.2.1`
- Kept `EslReactRuntimeRunnerAdapter` as the default `RuntimeRunnerInterface` binding
- Preserved real upstream lifecycle snapshot observation from `RuntimeRunnerHandle::lifecycleSnapshot()`
- Extended `EslReactRuntimeBootstrapInputFactory` to pass explicit dial URIs for non-default resolved transports while leaving the default TCP path unchanged
- Preserved package boundaries: reconnect, heartbeat, session lifecycle, prepared connector dial execution, and supervision remain owned by `apntalk/esl-react`

## Recommended release framing

Use this release as the tag/release description baseline:

> `0.4.1` finalizes the richer prepared dial-target extension of the Laravel-side runtime observation checkpoint for `apntalk/laravel-freeswitch-esl`. The package continues to bind the real `apntalk/esl-react` runner and consume `RuntimeRunnerHandle::lifecycleSnapshot()` for operator-facing status surfaces, and now also passes explicit prepared dial targets such as `tls://host:port` through the upstream runner input without taking ownership of reconnect, heartbeat, or runtime supervision.

## Deferred to later phases

- direct `apntalk/esl-core` transport polling handoff once public upstream seams exist
- live TLS validation against a real FreeSWITCH endpoint
- richer Laravel operator modeling around upstream lifecycle signals
- `0.5.x`: `apntalk/esl-replay` integration
