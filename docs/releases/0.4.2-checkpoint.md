# 0.4.2 Checkpoint Release Notes

## Release posture

`apntalk/laravel-freeswitch-esl` `0.4.2` is a truthful Laravel-side runtime observation checkpoint.
It extends the current `0.4.x` milestone with push-based lifecycle observation from `apntalk/esl-react`.
It is not a Laravel-owned live runtime milestone.

This checkpoint now includes:
- Laravel/control-plane foundations
- partial `apntalk/esl-core` integration
- real `apntalk/esl-react` runner binding
- real upstream lifecycle snapshot observation through `RuntimeRunnerHandle::lifecycleSnapshot()`
- push-based lifecycle observation through `RuntimeRunnerHandle::onLifecycleChange()`
- explicit prepared dial-target handoff through the Laravel-to-`apntalk/esl-react` adapter when the resolved `ConnectionContext` requires it

This checkpoint still does not include:
- Laravel-owned runtime supervision
- Laravel-owned reconnect/backoff ownership
- Laravel-owned heartbeat/session lifecycle ownership
- direct `apntalk/esl-core` transport polling handoff
- replay runtime orchestration
- live runtime validation against a real FreeSWITCH environment

## Highlights

- Bumped the minimum supported `apntalk/esl-react` line to `^0.2.2`
- Kept `EslReactRuntimeRunnerAdapter` as the default `RuntimeRunnerInterface` binding
- Preserved real lifecycle snapshot observation from `RuntimeRunnerHandle::lifecycleSnapshot()`
- Added push-based lifecycle observation by subscribing to `RuntimeRunnerHandle::onLifecycleChange()`
- Exposed push-observed lifecycle truth through Laravel worker status metadata and `freeswitch:worker` summary output
- Preserved package boundaries: reconnect, heartbeat, session lifecycle, prepared connector dial execution, and supervision remain owned by `apntalk/esl-react`

## Recommended release framing

Use this release as the tag/release description baseline:

> `0.4.2` finalizes the push-based lifecycle observation extension of the Laravel-side runtime observation checkpoint for `apntalk/laravel-freeswitch-esl`. The package continues to bind the real `apntalk/esl-react` runner and consume `RuntimeRunnerHandle::lifecycleSnapshot()` for operator-facing status surfaces, and now also subscribes to `RuntimeRunnerHandle::onLifecycleChange()` for ordered push-based lifecycle updates without taking ownership of reconnect, heartbeat, or runtime supervision.

## Deferred to later phases

- direct `apntalk/esl-core` transport polling handoff once public upstream seams exist
- richer Laravel operator modeling around upstream lifecycle transitions
- live runtime validation against a real FreeSWITCH endpoint
- `0.5.x`: `apntalk/esl-replay` integration
