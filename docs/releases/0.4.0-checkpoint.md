# 0.4.0 Checkpoint Release Notes

## Release posture

`apntalk/laravel-freeswitch-esl` `0.4.0` is a truthful Laravel-side lifecycle observation checkpoint.
It is not a Laravel-owned live runtime milestone.

This checkpoint now includes:
- Laravel/control-plane foundations
- partial `apntalk/esl-core` integration
- real `apntalk/esl-react` runner binding
- real upstream lifecycle snapshot observation through `RuntimeRunnerHandle::lifecycleSnapshot()`
- truthful worker, supervisor, and operator reporting for the supported upstream lifecycle fields

This checkpoint still does not include:
- Laravel-owned runtime supervision
- Laravel-owned reconnect/backoff ownership
- Laravel-owned heartbeat/session lifecycle ownership
- TLS prepared handoff through this runner path
- direct `apntalk/esl-core` transport polling handoff
- replay runtime orchestration

## Highlights

- Bumped the supported `apntalk/esl-react` line to `^0.2`
- Kept `EslReactRuntimeRunnerAdapter` as the default `RuntimeRunnerInterface` binding
- Switched Laravel status reporting from compatibility-only lifecycle support to real upstream `RuntimeRunnerHandle::lifecycleSnapshot()` consumption
- Hardened `WorkerStatus`, `WorkerSupervisor`, and `freeswitch:worker` reporting around real upstream connection/session/liveness/reconnect/drain truth
- Preserved package boundaries: reconnect, heartbeat, session lifecycle, and supervision remain owned by `apntalk/esl-react`

## Recommended release framing

Use this release as the tag/release description baseline:

> `0.4.0` finalizes the first truthful Laravel-side lifecycle observation checkpoint for `apntalk/laravel-freeswitch-esl`. The package now binds the real `apntalk/esl-react` runner by default and consumes `RuntimeRunnerHandle::lifecycleSnapshot()` end to end for operator-facing status surfaces, while reconnect, heartbeat, session lifecycle, and supervision remain owned upstream.

## Deferred to later phases

- richer Laravel operator modeling around upstream lifecycle signals
- TLS/direct transport runner handoff once public upstream seams exist
- `0.5.x`: `apntalk/esl-replay` integration
