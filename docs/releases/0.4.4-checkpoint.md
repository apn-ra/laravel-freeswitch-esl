# 0.4.4 Checkpoint Release Notes

## Release posture

`apntalk/laravel-freeswitch-esl` `0.4.4` is a truthful Laravel-side dependency-alignment checkpoint.
It aligns the package to the released `apntalk/esl-react` `^0.2.5` runtime/lifecycle line.
It is not a Laravel-owned live runtime milestone.

This checkpoint includes:
- Laravel/control-plane foundations
- partial `apntalk/esl-core` integration
- real `apntalk/esl-react` runner binding
- lifecycle snapshot observation through `RuntimeRunnerHandle::lifecycleSnapshot()`
- push-based lifecycle observation through `RuntimeRunnerHandle::onLifecycleChange()`
- richer prepared dial-target handoff through the Laravel-to-`apntalk/esl-react` adapter
- explicit dependency alignment to `apntalk/esl-react` `^0.2.5`

This checkpoint still does not include:
- Laravel-owned reconnect/backoff behavior
- Laravel-owned heartbeat/session lifecycle ownership
- Laravel-owned bgapi/event runtime semantics
- Laravel-owned runtime supervision
- direct `apntalk/esl-core` transport polling handoff
- replay runtime orchestration
- live runtime validation from this Laravel repository

## Highlights

- Bumped the minimum supported `apntalk/esl-react` line to `^0.2.5`
- Kept `EslReactRuntimeRunnerAdapter` as the default `RuntimeRunnerInterface` binding
- Preserved lifecycle snapshot and push-based lifecycle observation consumption
- Preserved richer prepared dial-target handoff behavior
- Documented that combined bgapi/event activity under degraded liveness or reconnecting states is upstream-owned and upstream-validated in `apntalk/esl-react`
- Preserved package boundaries: Laravel reports upstream lifecycle truth through worker/operator surfaces but does not own reconnect, heartbeat, session lifecycle, bgapi/event runtime behavior, prepared connector dial execution, or supervision

## Recommended release framing

Use this release as the tag/release description baseline:

> `0.4.4` aligns `apntalk/laravel-freeswitch-esl` to the released `apntalk/esl-react` `^0.2.5` runtime/lifecycle line. The package continues to bind the real upstream runner, consume lifecycle snapshots and push-based lifecycle updates for worker/operator status, and pass richer prepared dial targets through the upstream runner input. Combined bgapi/event activity, degraded liveness, reconnecting behavior, heartbeat/session lifecycle, and runtime supervision remain owned and validated upstream.

## Deferred to later phases

- direct `apntalk/esl-core` transport polling handoff once public upstream seams exist
- richer Laravel operator modeling around upstream lifecycle transitions
- live runtime validation from this Laravel repository
- `0.5.x`: `apntalk/esl-replay` integration
