# 0.6.x Private Live Validation Runbook

## Purpose

This runbook describes one bounded, read-only validation pass against a private
FreeSWITCH environment for the `0.6.x` release line.

Use it only after deterministic repo-owned evidence is already green:

- `composer test`
- `vendor/bin/phpstan analyse --no-progress`
- `vendor/bin/pint --test`

This is operator-run environment validation. It is not a public-CI gate and it
does not change package ownership boundaries.

## What this runbook proves

This runbook proves only the following bounded checks:

- TCP reachability to the target ESL endpoint
- inbound ESL auth acceptance
- one read-only `api status` round-trip
- one bounded event subscription/bootstrap in `event-plain`
- one bounded event subscription/bootstrap in `event-json`

## What this runbook does not prove

- dialplan behavior
- media or call flow
- reconnect ownership
- heartbeat/session lifecycle ownership
- replay execution or replay re-injection
- long-running production soak behavior

## Safety posture

- run only against a known-safe validation target
- keep the scope read-only
- do not dial, bridge, originate, or execute telephony actions
- do not store secrets in shell history, committed files, or shared logs
- do not commit raw validation artifacts to the repository

## Prerequisites

Before running the validation:

- the RC ref under review is checked out locally
- the operator has private-network reachability to the FreeSWITCH target
- the target allows inbound ESL auth and read-only event subscription bootstrap
- the operator has the target host, port, and password through an approved
  secret-handling path
- PHP dependencies are installed locally with `composer install`

Recommended shell posture:

```bash
git fetch origin --tags
git checkout main
git pull --ff-only origin main
composer install --prefer-dist --no-interaction --no-progress
```

## Environment template

Copy the dedicated template first:

```bash
cp .env.private-validation.example .env.private-validation
```

Fill in only the private values required for the target environment. Do not
commit the filled-in copy.

## Required inputs

The wrapper reads `.env.private-validation` by default, or another file passed
via `--env-file=...`.

Relative env-file paths and relative artifact output paths are resolved from the
repository root so the wrapper behaves predictably even if the operator invokes
it from another working directory.

Required variables:

- `FREESWITCH_ESL_SMOKE_HOST`
- `FREESWITCH_ESL_SMOKE_PORT`
- `FREESWITCH_ESL_SMOKE_PASSWORD`

Optional variables:

- `FREESWITCH_ESL_SMOKE_READ_ONLY=true`
- `FREESWITCH_ESL_SMOKE_TIMEOUT=8`
- `FREESWITCH_ESL_SMOKE_CAPTURE_FRAMES=true`
- `FREESWITCH_ESL_SMOKE_OUTPUT_DIR=build/live-smoke-review`

Do not write real values into committed docs, tracked config, or release notes.

## Validation commands

Primary operator entrypoint:

```bash
php bin/freeswitch-private-live-validate.php --env-file=.env.private-validation
```

Useful preflight:

```bash
php bin/freeswitch-private-live-validate.php \
  --env-file=.env.private-validation \
  --dry-run
```

The dry-run path performs configuration, helper-availability, and artifact-path
preflight without making a network connection.

The wrapper runs the four bounded checks in order using the upstream
`vendor/apntalk/esl-core/tools/smoke/live_freeswitch_readonly_validate.php`
helper and writes artifacts into the configured output directory.
The current direct-helper path relies on the upstream bootstrap fix shipped in
`apntalk/esl-core v0.2.8`; no downstream proxy/workaround is required.

## Expected success signals

Treat the run as successful only if all of the following are true:

- each command exits `0`
- `auth.json` shows successful connect/auth
- `api-status.json` shows successful `api status`
- `event-plain.json` shows successful event bootstrap in plain mode
- `event-json.json` shows successful event bootstrap in JSON mode
- the review directory contains the four JSON files
- no dialing, media, or destructive actions were performed

## Expected retained artifacts

Retain locally for release review:

- `build/live-smoke-review/auth.json`
- `build/live-smoke-review/api-status.json`
- `build/live-smoke-review/event-plain.json`
- `build/live-smoke-review/event-json.json`
- `build/live-smoke-review/summary.json`
- `build/live-smoke-review/release-review.md`
- optional raw frames under `build/live-smoke-review/captures/`

Do not commit these files to the repository.

## Failure handling

Fail closed if any of the following occur:

- target unreachable
- auth failure
- `api status` failure
- event bootstrap failure in either mode
- incomplete or missing evidence files
- evidence suggests anything beyond the intended read-only scope

If the run fails:

- classify it as environment-owned, secret-owned, repo-owned, or upstream-owned
- record the exact failing step and local timestamp
- keep the wrapper stderr and any `*.stderr.log` files alongside the retained
  evidence directory for review
- do not claim RC promotion readiness from that run

## Recording the outcome

After a successful run, prepare an environment-safe evidence summary for release
review:

- ref and commit validated
- execution timestamp in UTC
- validation path used: `private host` or `private runner`
- retained evidence file set
- pass/fail result for `auth`, `api status`, `event-plain`, `event-json`
- confirmation that the scope remained read-only

Suggested summary template:

```md
# 0.6.0 RC2 Live Smoke Evidence

- Validation path: `private host`
- Ref: `main`
- Commit: `<HEAD_SHA>`
- Executed at: `<UTC_TIMESTAMP>`
- Evidence: `build/live-smoke-review/`

## Bounded checks

- `auth`: pass
- `api status`: pass
- `event-plain`: pass
- `event-json`: pass

## Scope confirmation

- read-only validation only
- no dialing or destructive actions performed
```

Record only environment-safe summaries. Do not include hostnames, passwords,
IPs, or raw frame contents in committed release docs.
