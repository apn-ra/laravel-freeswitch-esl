# 0.6.0 RC1 Release Notes

## Release posture

`apntalk/laravel-freeswitch-esl` `0.6.0-rc1` is a bounded release candidate for
the `0.6.x` hardening line.
It prepares the package for a truthful `0.6.x` tag without taking reconnect,
heartbeat/session lifecycle, replay execution, or protocol ownership away from
`apntalk/esl-react`, `apntalk/esl-replay`, or `apntalk/esl-core`.

This release candidate now includes:

- truthful authority docs aligned to the shipped architecture and current
  upstream delegation boundaries
- PHP `8.3+` as an explicit support floor across Composer metadata and repo docs
- shipped non-null metrics drivers, with the log-backed recorder as the default
  package binding and event/null as explicit alternatives
- load-bearing `max_inflight` enforcement in `WorkerRuntime`, including
  fail-closed rejection while draining or when the configured limit is exceeded
- additive worker/backpressure status metadata such as
  `max_inflight`, `backpressure_active`, `backpressure_limit_reached`,
  `backpressure_reason`, and `backpressure_rejected_total`
- deterministic simulated ESL lifecycle verification through the current
  `apntalk/esl-react` package boundary
- a near-runnable example-app cookbook under `examples/laravel-app`
- clean PHPStan and default PHPUnit gates suitable for RC evidence
- an optional manual live smoke workflow for read-only environment validation

This release candidate still does not include:

- Laravel-owned reconnect/backoff mechanics
- Laravel-owned heartbeat/session lifecycle ownership
- Laravel-owned replay execution or session recovery
- protocol parsing or low-level transport ownership
- `1.0.0` stability claims

## Upgrade notes

- PHP `8.3+` is now the truthful floor for the current release line. Downstream
  applications pinned to PHP `8.2` must not upgrade to `0.6.x`.
- Default metrics behavior changed. The package no longer binds
  `MetricsRecorderInterface` to an implicit no-op in default installs; it now
  uses the shipped `log` recorder unless
  `freeswitch-esl.observability.metrics.driver` is set to `event` or `null`.
- `freeswitch-esl.drain_defaults.max_inflight` is now enforced instead of merely
  reported. Installations that previously configured the value but did not see
  runtime rejection behavior should expect fail-closed refusal when the limit is
  exceeded or a worker is draining.
- Machine-readable worker status and worker-derived health metadata now expose
  bounded backpressure and rejection fields. Downstream automation can consume
  them without inferring queue ownership or dead-letter behavior.
- `examples/laravel-app` is now a cookbook for adoption validation. It is still
  not a shipped full Laravel application skeleton.

## Verification evidence

- `vendor/bin/phpstan analyse --no-progress`
- `vendor/bin/phpunit --configuration phpunit.xml`
- `composer test`
- `vendor/bin/pint --test`
- deterministic lifecycle verification in
  `tests/Integration/Runtime/WorkerRuntimeEslReactLifecycleTest.php`

The optional live smoke workflow remains outside the default CI path. It is for
environment validation, not for deterministic package release gating.

## Optional live smoke workflow

The repository now ships a manual GitHub Actions workflow at
`.github/workflows/live-smoke.yml`.

Workflow posture:

- manual only via `workflow_dispatch`
- secrets-gated and skip-clean when required environment values are absent
- non-destructive and read-only
- reuses the upstream non-public validator in
  `vendor/apntalk/esl-core/tools/smoke/live_freeswitch_readonly_validate.php`
  rather than introducing new live-runtime ownership in this package

What it proves:

- TCP connect to the configured ESL endpoint
- auth acceptance
- one bounded `api status` round-trip
- one bounded event subscription/bootstrap path in `event plain` and `event json`
  modes

What it does not prove:

- dialplan behavior
- media/call flow
- worker recovery ownership
- replay execution

## Recommended release framing

Use this release as the tag/release description baseline:

> `0.6.0-rc1` is a bounded release candidate for the `0.6.x` hardening line of
> `apntalk/laravel-freeswitch-esl`. It aligns package docs to shipped ownership,
> makes metrics and inflight backpressure load-bearing for default installs,
> strengthens lifecycle verification with a deterministic ESL harness, and adds
> a manual read-only live smoke workflow. Reconnect, heartbeat/session lifecycle,
> protocol internals, and replay execution remain owned upstream.

## Deferred after RC

- live smoke evidence from a real environment
- any broader release automation around manual smoke artifacts
- `1.0.0` stabilization work
