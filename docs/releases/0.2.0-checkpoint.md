# 0.2.0 Checkpoint Release Notes

## Release posture

`apntalk/laravel-freeswitch-esl` `0.2.0` is a truthful partial `0.2.x` Laravel-side integration checkpoint.
It is not a live runtime milestone.

This checkpoint now includes:
- Laravel/control-plane foundations
- partial `apntalk/esl-core` integration
- stable upstream transport/bootstrap seam usage
- Laravel-side connection/runtime handoff scaffolding
- truthful non-live worker/runtime status surfaces

This checkpoint still does not include:
- live async runtime loops
- reconnect/backoff supervision
- heartbeat/session lifecycle
- listener/runtime ownership
- replay-engine/runtime orchestration

## Highlights

- Updated the package to use stable public `apntalk/esl-core` seams:
  - `TransportFactoryInterface`
  - `SocketTransportFactory`
  - `SocketEndpoint`
  - `InboundPipeline::withDefaults()`
  - `InboundConnectionFactoryInterface`
- Added and bound the Laravel-side connection-factory/handoff seam around `ConnectionContext` and `EslCoreConnectionHandle`
- Hardened worker/runtime scaffolding so handoff preparation is retained and surfaced through status and command paths without overstating runtime activity
- Tightened package-boundary, public API, and compatibility docs to match the current checkpoint truthfully
- Passed contract, unit, integration, and static-analysis verification for the checkpoint work

## Recommended release framing

Use this release as the tag/release description baseline:

> `0.2.0` finalizes the first Laravel-side `apntalk/esl-core` integration checkpoint for `apntalk/laravel-freeswitch-esl`. The package now ships stable upstream seam usage, connection/runtime handoff scaffolding, and truthful non-live worker status surfaces. Live runtime ownership remains deferred to `apntalk/esl-react`.

## Deferred to later phases

- `0.3.x`: `apntalk/esl-react` runtime wiring
- later hardening: reconnect, heartbeat, supervision maturity
- `0.5.x`: `apntalk/esl-replay` integration
