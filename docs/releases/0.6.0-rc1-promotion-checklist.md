## How to run the manual live smoke workflow

From a local clone with GitHub CLI access:

```bash
git fetch origin --tags
git checkout main
git pull --ff-only origin main

gh workflow run "Live Smoke" \
  --ref main \
  -f timeout_seconds=8 \
  -f capture_frames=true
```

To locate the latest run:

```bash
gh run list --workflow "Live Smoke" --limit 5 \
  --json databaseId,displayTitle,headBranch,headSha,status,conclusion,createdAt,updatedAt,url
```

To wait for completion:

```bash
gh run watch <RUN_ID> --exit-status
```

To inspect logs:

```bash
gh run view <RUN_ID> --log
```

To inspect artifact metadata:

```bash
gh api repos/apn-ra/laravel-freeswitch-esl/actions/runs/<RUN_ID>/artifacts
```

To download the artifacts locally:

```bash
mkdir -p build/live-smoke-review
gh run download <RUN_ID> --dir build/live-smoke-review
```

## Required artifact set

After a real smoke run, the workflow should upload one artifact bundle under a
name like:

- `live-smoke-<RUN_ID>`

That bundle should contain:

- `auth.json`
- `api-status.json`
- `event-plain.json`
- `event-json.json`
- optional raw-frame captures under `captures/`

If the artifact is missing entirely, treat that as a failed smoke run.

## Pass/fail criteria

Promotion is supported only if all of the following are true:

- the workflow did not skip the smoke steps for missing secrets
- `auth.json` indicates successful connect/auth
- `api-status.json` indicates successful `api status`
- `event-plain.json` indicates successful event subscription/bootstrap in plain
  mode
- `event-json.json` indicates successful event subscription/bootstrap in JSON
  mode
- artifacts are downloadable and inspectable
- no dialing, media, or destructive telephony actions were performed

Fail closed if any of the following occur:

- missing secrets
- connect/auth failure
- `api status` failure
- event bootstrap failure in either format
- no artifact upload
- evidence suggests anything beyond the intended read-only validation scope

## Evidence handling rules

When reviewing or recording the smoke result:

- do not commit raw artifacts to the repository
- do not record hostnames, passwords, IPs, or sensitive payloads in docs
- record only environment-safe summaries, run IDs, artifact names, timestamps,
  and bounded pass/fail results

## Tag and release commands after smoke passes

If the RC tag does not yet exist locally:

```bash
git fetch origin --tags
git checkout main
git pull --ff-only origin main
git tag -a v0.6.0-rc1 -m "0.6.0 RC1"
```

If the RC tag already exists locally, verify it points at the intended commit:

```bash
git rev-list -n 1 v0.6.0-rc1
git rev-parse origin/main
```

If they match, push the tag:

```bash
git push origin v0.6.0-rc1
```

If you added an evidence doc or release-surface follow-up commit before release,
push the branch first:

```bash
git add docs/releases/0.6.0-rc1-live-smoke-evidence.md docs/releases/0.6.0-rc1.md CHANGELOG.md
git commit -m "Record 0.6.0 RC1 live smoke evidence"
git push origin main
```

Then create a draft GitHub release:

```bash
gh release create v0.6.0-rc1 \
  --draft \
  --title "0.6.0 RC1" \
  --notes-file docs/releases/0.6.0-rc1.md
```
