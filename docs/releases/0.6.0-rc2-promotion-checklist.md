## Required artifact set

After a real RC2 validation run, the evidence set should contain:

- `auth.json`
- `api-status.json`
- `event-plain.json`
- `event-json.json`
- `summary.json`
- `release-review.md`
- optional raw-frame captures under `captures/` when capture is enabled

If using the helper workflow, the uploaded artifact bundle will typically be
named `live-smoke-<RUN_ID>`. If using direct operator-run validation, keep the
same file names in a local review directory and do not commit them.

## Pass/fail criteria

Promotion is supported only if all of the following are true:

- the chosen validation path completed a real read-only validation run for the
  exact RC2 ref
- `auth.json` indicates successful connect/auth
- `api-status.json` indicates successful `api status`
- `event-plain.json` indicates successful event subscription/bootstrap in plain
  mode
- `event-json.json` indicates successful event subscription/bootstrap in JSON
  mode
- `summary.json` and `release-review.md` are present
- evidence files are inspectable by the operator
- no dialing, media, or destructive telephony actions were performed

Fail closed if any of the following occur:

- missing private-environment credentials or reachability
- connect/auth failure
- `api status` failure
- event bootstrap failure in either format
- no inspectable evidence files
- evidence suggests anything beyond the intended read-only validation scope
